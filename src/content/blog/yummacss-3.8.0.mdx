**This update involves significant breaking changes across the codebase. We recommend upgrading with caution and thoroughly testing your implementation, as existing features will likely break.**

---

## Monorepo transition

The Yumma CSS ecosystem has transitioned to a monorepo. The `api`, `nitro` (the Yumma CSS engine), & `runtime` now reside in the main [yummacss](https://github.com/yummacss/yummacss) repository.

This consolidation improves development efficiency & ensures package synchronization.

### Public archives

Repositories for API, Nitro, & Runtime are now archived. Future development occurs in the [main monorepo](https://github.com/yummacss/yummacss).

---

## Version synchronization

Core package versions are synchronized with the main Yumma CSS version:

- `@yummacss/api@1.8.0` -> `@yummacss/api@3.8.0`
- `@yummacss/nitro@0.5.0` -> `@yummacss/nitro@3.8.0`
- `@yummacss/runtime@0.3.0` -> `@yummacss/runtime@3.8.0`

---

## No API changes

This update is a structural change. Existing functionalities, bug fixes, & usage patterns remain unchanged.

---

## Upgrade

Get the latest version of Yumma CSS:

```bash
pnpm up yummacss@latest
```
