## What's new in v3.0?

We've completely rewritten the code base for Yumma CSS v3.0, both internally and externally, and we're excited to improve the way you deal with CSS.

In v3.0, we're planning to address some major performance issues and add and improve existing utilities in the framework, among other things.

### All-new CLI

Until recently, you had to import a lot of annoying CSS from Yumma CSS, which was a real pain.

With v3.0, you won't have to stress about shipping unused CSS to the browser. The new CLI will scan and get rid of them for you automatically.

<Stepper>
  <Step title="Install Yumma CSS">
    Add `yummacss` to your project as dev dependency.

    <Note icon="info">Learn more about [dependency changes](#dependency-changes).</Note>

    ```bash
    npm install yummacss -D
    ```
  </Step>

  <Step title="Add the configuration file">
    Next, add the `yumma.config.js` to the root level of your project or run `npx yummacss init` to create it for you.

    ```
      - node_modules/
      - public
        - favicon.ico
      - src
        - out.css
        - index.html
      - .gitignore
      - package-lock.json
      - package.json
      - **yumma.config.js**
    ```
  </Step>

  <Step title="Set up the config file">
    To generate styles using the CLI, just set up the `source` array with the path to your template files and set the `output` string with the path to the CSS file you want to generate.

    ```js title="yumma.config.js" mark={2-3}
    module.exports = {
      source: ["./src/**/*.html"],
      output: "./src/out.css",
      buildOptions: {
        reset: true,
        minify: false,
      },
    };
    ```
  </Step>

  <Step title="Write CSS">
    Start using Yumma CSS utilities in your to generate CSS with the CLI.

    ```html mark={7, 11-16} title="index.html"
    <!DOCTYPE html>
    <html lang="en">

    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <link rel="stylesheet" href="/src/out.css" />
    </head>

    <body>
      <div class="b-1 bc-silver-2 bg-white rad-2 bs-sm p-4">
          <h1 class="fw-600 tc-indigo fs-lg">Hello 👋, name's Renildo.</h1>
          <p class="tc-gray-7">I'm the Founder of Yumma CSS!</p>

          <button class="bg-indigo fs-sm h:bg-indigo-7 mt-6 px-4 py-1 tc-white">GitHub</button>
      </div>
    </body>

    </html>
    ```
  </Step>

  <Step title="Compile the SCSS">
    To compile the source into CSS, ou can run the following command:

    ```bash
    npx yummacss build
    ```

    When you run the `build` command the CLI will create a new CSS file and scan project paths based on the `yumma.config.js` to purge of any unused styles.

    ```css title="out.css"
    .rad-1 {
      border-radius: .25rem;
    }

    .rad-2 {
      border-radius: .5rem;
    }

    .b-1 {
      border-width: 1px;
    }

    .mt-6 {
      margin-top: 1.5rem;
    }

    .p-4 {
      padding: 1rem;
    }

    .px-4 {
      padding-left: 1rem;
      padding-right: 1rem;
    }

    .py-1 {
      padding-bottom: .25rem;
      padding-top: .25rem;
    }

    /* etc */
    ```
  </Step>
</Stepper>

### Build performance

We completely overhauled the entire codebase to get better performance in build times and overall file size. We changing the way all utilities are generated, to eliminate any potential for duplicated or unnecessary data in the `/dist` folder.

| Metric               | v2      | v3.0      | Improvement            |
| -------------------- | ------- | ------- | ---------------------- |

| Utilities coverage   | 111     | 167     | +56                    |

### Upgrading to v3.0

If you're using Yumma CSS v2 or older, the process of upgrading to v3.0 using the Yumma CLI is honestly super easy.

<Stepper>
  <Step title="Remove @import rules">
    The CLI works by compiling SCSS to CSS, so there's no need to import the Yumma CSS package dependency.

    ```css title="globals.css" del={1}
    @import "/node_modules/yummacss/dist/yumma.min.css";
    ```
  </Step>

  <Step title="Add the Yumma config file">
    Create a `yumma.config.js` file at the project root.

    ```
    ├── node_modules/
    ├── public/
    │   └── favicon.ico
    ├── src/
    │   ├── globals.css
    │   └── index.html
    ├── .gitignore
    ├── package-lock.json
    ├── package.json
    └── yumma.config.js <-
    ```
  </Step>

  <Step title="Set up the config file">
    Setup the `source` array and `output` string field.

    ```js title="yumma.config.js" mark={2-7}
    module.exports = {
      source: ["./src/**/*.html"],
      output: "./src/globals.css",
      buildOptions: {
        reset: true,
        minify: true,
      },
    };
    ```
  </Step>

  <Step title="Compile the SCSS">
    Run `npx yummacss build` in your terminal to compile to CSS.

    ```bash
    npx yummacss build
    ```
  </Step>
</Stepper>

## Changes in v3.0

Like some of the past major updates, Yumma CSS v3.0 is changing its core framework structure. Just so you know, there are going to be a few breaking changes in the release, so it might be worth taking a look before you update.

### Dependency changes

With Yumma CSS v3.0 you no longer need to install `yummacss` as a standard dependency. Instead, you can install it as a dev dependency. This is because Yumma CSS is now a CLI tool that generates CSS files based on your configuration.

```bash
npm install yummacss -D
```

The full set of utilities will be retained within the distribution folder for the purpose of facilitating the importation of the entire Yumma utilities suite, should this be required.

```
- dist
 	- yumma.css
 	- yumma.min.css
```

---

### Color utility changes

In v3.0, both the light (`l-`) and dark (`d-`) characters are being removed across all color utilities. As a result, the range used to determine a color's shade was also adjusted.

<div className="p-4">
  ```html title="index.html" del={1} ins={2}
  <button class="bg-l-indigo-6 h:bg-d-indigo-1">Button</button>
  <button class="bg-indigo-1 h:bg-indigo-7">Button</button>
  ```
</div>

Also, the color hue is increasing from `10%` shade modification to `14%`. This means that light colors will become lighter, and dark colors will become darker.

| *-l-indigo-2 | *-indigo-5  | -2 characters |
| *-l-indigo-1 | *-indigo-6  | -2 characters |

### Base style changes

We're making some changes to the base styles in v3.0 to make it more modern and consistent. These changes are turned on by default, but you can turn them off using the config file.

By default, all paddings will be removed.

```scss mark={3} title="base/stylecent.scss"
* {
  margin: 0;
  padding: 0;
}
```

Font family is set in the `<html>` tag element. - [oshwcomeau.com](https://joshwcomeau.com/css/custom-css-reset)

```scss mark={2} title="base/stylecent.scss"
html {
  font-family: vars.$yma-font-system;
}
```

Font rendering is smoother, and a consistent system font is set as the default. - [oshwcomeau.com](https://joshwcomeau.com/css/custom-css-reset)

```scss ins={2-3} del={4} title="base/stylecent.scss"
body {
  -webkit-font-smoothing: antialiased;
  font-family: vars.$yma-font-system;
  line-height: 1.5;
}
```

Form elements now include padding by default. Borders are added for form elements without class attributes.

```scss mark={6-8,16} title="base/stylecent.scss"
button,
input,
optgroup,
select,
textarea {
  background-color: vars.$yma-color-transparent;
  font-family: inherit;
  padding: 0.5rem;
}

button:not([class]),
input:not([class]),
optgroup:not([class]),
select:not([class]),
textarea:not([class]) {
  border: 1px solid vars.$yma-color-silver;
}
```

Interactive elements have clear outlines for accessibility.

```scss mark={7-9} title="base/stylecent.scss"
button,
input,
textarea,
select,
a,
summary {
  &:focus {
    outline: 2px solid vars.$yma-color-transparent;
  }
}
```

In the absence of content, textareas will exhibit a default height. - [piccalil.li](https://piccalil.li/blog/a-more-modern-css-reset)

```scss mark={2} title="base/stylecent.scss"
textarea:not([rows]) {
  min-height: 10em;
}
```

Disabled elements are visually distinct with reduced opacity and a "not-allowed" cursor.

```scss mark={5-6} title="base/stylecent.scss"
button:disabled,
input:disabled,
select:disabled,
textarea:disabled {
  cursor: not-allowed;
  opacity: 0.5;
}
```

Headings adopt balanced text wrapping, consistent font sizes, and bold weights. - [oshwcomeau.com](https://joshwcomeau.com/css/custom-css-reset)

```scss mark={7-9,13} title="base/stylecent.scss"
h1,
h2,
h3,
h4,
h5,
h6 {
  font-size: 1rem;
  font-weight: 600;
  text-wrap: balance;
}

p {
  text-wrap: pretty;
}
```

Small text and code elements are consistently scaled and inherited. Code elements will have consistent font family. - [modern-normalize](https://github.com/sindresorhus/modern-normalize/blob/main/modern-normalize.css#L64)

```scss mark={3,7-8,15-16} title="base/stylecent.scss"
b,
strong {
  font-weight: 700;
}

small {
  font-size: 80%;
  line-height: 1.4;
}

pre,
code,
kbd,
samp {
  font-family: monospace, monospace;
  font-size: 1em;
}
```

Reset default link styles.

```scss mark={2,7} title="base/stylecent.scss"
a {
  color: inherit;
  text-decoration: none;
}
```

Table headers are bold and sized consistently.

```scss mark={2-3} title="base/stylecent.scss"
th {
  font-size: 1rem;
  font-weight: 600;
}
```

Horizontal rules, details, and summaries are updated for better spacing and display. - [modern-normalize](https://github.com/sindresorhus/modern-normalize/blob/main/modern-normalize.css#L64)

```scss mark={3,8,12} title="base/stylecent.scss"
hr {
  border-top: 1px solid vars.$yma-color-silver;
  height: 0;
  margin: 1em 0;
}

details {
  display: block;
}

summary {
  display: list-item;
}
```

#### Disabling base styles

To disable the base styles generation you can set `buildOptions.reset` to `false` in the `yummacss.config.mjs` file.

```js mark={5} title="yummacss.config.mjs"
export default {
  source: ["./src/**/*.html"],
  output: "./src/out.css",
  buildOptions: {
    reset: false, // disables base styles
    minify: false,
  },
};
```

### Breakpoint changes

In v3.0, we are moving from pixel-based units to rem-based units. This change promotes better accessibility and scalability, as rem units respect the user's browser font size settings.

| v2     | v3.0               | Breakpoint |
| ------ | ---------------- | ---------- |
| 640px  | 40rem _(640px)_  | sm         |
| 768px  | 48rem _(768px)_  | md         |
| 1024px | 64rem _(1024px)_ | lg         |
| 1280px | 80rem _(1280px)_ | xl         |
| 1536px | 96rem _(1536px)_ | xxl        |

### Fixed media query utilities

We've grouped related media query utilities together so they can override existing ones in the DOM like they were supposed to. This improvement also eliminates redundant CSS classes and reduces the overall file size.
