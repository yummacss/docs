## Arbitrary chaining support

Chaining variants to pseudo-classes or using opacity modifiers with active, focus, or hover wasn't possible before. You can now link variants to whatever variants you like.

| Class              | Generated CSS Rule                                                      |
|--------------------|-------------------------------------------------------------------------|
| `bg-red`           | `.bg-red { ... }`                                                       |
| `bg-red/50`        | `.bg-red\/50 { ... }`                                                   |
| `h:bg-red/50`      | `.h\:bg-red\/50:hover { ... }`                                          |
| `lg:h:bg-red/50`   | `@media (width >= 64rem) { .lg\:h\:bg-red\/50:hover { ... } }`          |
| `lg:h:f:bg-red/25` | `@media (width >= 64rem) { .lg\:h\:f\:bg-red\/25:hover:focus { ... } }` |

### Opacity modifiers with hover

<Hint icon="cursor">Try hovering the div to see how the opacity changes.</Hint>

```html
<div class="bg-indigo-6/25 h:bg-indigo-6/50 d-16"></div>
```

### Media variants with pseudo-classes

<Hint icon="size">Try resizing the window to see how the opacity changes when hovered.</Hint>

```html
<div class="bg-indigo-6/25 lg:h:bg-indigo/50 h:bg-indigo/25 d-16"></div>
```

## Tokenizer improvements

This version also tweaked the regular expressions to correctly capture complex class names with colons and slashes, especially within template literals.

```ts
// Tokenizer now correctly extracts:
const tokens = tokenizer('const x = "lg:h:bg-red/50"');
// Result: ['lg:h:bg-red/50']
```
