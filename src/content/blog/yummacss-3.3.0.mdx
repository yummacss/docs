**This update involves significant breaking changes across the codebase. We recommend upgrading with caution and thoroughly testing your implementation, as existing features will likely break.**

---

## Using .mjs configuration

Yumma CSS v3.3 transitions from `.js` to `.mjs` for configuration files, removing the need for `type="module"` in `package.json` & optimizing CLI module detection.

```diff
- yumma.config.js
+ yumma.config.mjs
```

---

## Configuration file generation

This release fixes JSON syntax issues in the configuration file generation. The `init` command now generates valid configuration properties.

```js title="yumma.config.mjs"
export default {
  source: [""],
  output: "",
  buildOptions: {
    reset: true,
    minify: false,
  },
};
```
