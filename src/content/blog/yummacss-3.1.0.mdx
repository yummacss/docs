## New pseudo-class variants

We're expanding the framework's interactivity capabilities by adding support for two essential pseudo-class variants that are compatible with all the framework's base utilities.

### Focus variants

You can now apply styles to elements that receive focus using the `f:` prefix. This is extremely useful for making form elements and buttons more accessible and interactive.

<Preview>
    <button className="bg-indigo f:oc-indigo-5 f:oo-2 f:os-s f:ow-2 h:bg-indigo-7 px-4 py-2 d-14"></button>
</Preview>

```html "f:oc-indigo-5 f:oo-2 f:os-s f:ow-2"
<button class="bg-indigo f:oc-indigo-5 f:oo-2 f:os-s f:ow-2 h:bg-indigo-7 px-4 py-2 d-14"></button>
```

### Active variants

The `a:` prefix allows you to style elements when they are being pressed or clicked. It's perfect for creating responsive button interactions and touch feedback.

<Preview>
    <div className="bg-indigo h:bg-indigo-7 a:bg-indigo-9 px-4 py-2 d-14"></div>
</Preview>

```html "a:bg-indigo-9"
<div class="bg-indigo h:bg-indigo-7 a:bg-indigo-9 px-4 py-2 d-14"></div>
```

## Legacy CSS files removed

As part of our ongoing efforts to improve performance and optimize Yumma CSS, we have completely removed the legacy CSS files, `yumma.css` and `yumma.min.css`, from the distribution directory.

### What does this mean?

If you're already using the Yumma CSS CLI ([introduced in v3.0](/blog/yummacss-3.0.0)), this change should not feel like a breaking change. However, if you're still importing CSS files directly, you will need to migrate.

Learn more about how to migrate to the CLI-based workflow [here](/docs/upgrading).

You can get the latest version of Yumma CSS from npm or pnpm:
