## New pseudo-class variants

Yumma CSS v3.1.0 adds support for focus & active pseudo-class variants compatible with all base utilities.

### Focus variants

Apply styles to elements that receive focus using the `f:` prefix, improving form & button accessibility.

<Preview>
    <button className="bg-indigo f:oc-indigo-5 f:oo-2 f:os-s f:ow-2 h:bg-indigo-7 px-4 py-2 d-14"></button>
</Preview>

```html "f:oc-indigo-5 f:oo-2 f:os-s f:ow-2"
<button class="bg-indigo f:oc-indigo-5 f:oo-2 f:os-s f:ow-2 h:bg-indigo-7 px-4 py-2 d-14"></button>
```

### Active variants

The `a:` prefix styles elements during interaction, perfect for button feedback & touch interactions.

<Preview>
    <div className="bg-indigo a:bg-indigo-9 px-4 py-2 d-14"></div>
</Preview>

```html "a:bg-indigo-9"
<div class="bg-indigo a:bg-indigo-9 px-4 py-2 d-14"></div>
```

## Legacy CSS files removed

Legacy `yumma.css` & `yumma.min.css` files are removed from the distribution directory for performance & optimization.

### Migration

If using the Yumma CSS CLI, this is not a breaking change. However, if importing CSS files directly, migration is required.

Learn more in the [upgrade guide](/docs/upgrading).
