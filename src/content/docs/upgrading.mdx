Modern versions of Yumma CSS introduce significant improvements, including a new CLI-based workflow & core shifts in utility naming.

Previous versions relied on `@import` statements; the current system uses a dedicated CLI for better performance & flexibility.

## Legacy versions

In this guide, **legacy** refers to Yumma CSS v2 & earlier versions. These older systems are no longer actively developed & lack the performance optimizations of the current CLI-driven core.

<Stepper>
  <Step title="Update Yumma CSS">
    Remove the old `yummacss` package.

    ```bash title="Terminal"
    pnpm add yummacss@latest
    ```
  </Step>

  <Step title="Initialize configuration">
    Create a configuration file in your project.

    ```bash title="Terminal"
    pnpx yummacss init
    ```
  </Step>

  <Step title="Delete old imports">
    Remove the old `@import` statements from your CSS files.

    ```css title="styles.css" del={1-2}
    @import "yummacss/dist/yumma.min.css";
    @import "yummacss/dist/yumma.css";
    ```
  </Step>

  <Step title="Set up configuration">
    Specify the locations of all your project files in the config file.

    ```js title="yumma.config.mjs" mark={2-3}
    export default {
      source: ["./src/**/*.html"],
      output: "./src/styles.css",
      buildOptions: {
        reset: true,
        minify: false,
      },
    };
    ```
  </Step>

  <Step title="Build styles">
    You can now start generating your CSS with the [`build`](/docs/configuration/#build) or [`watch`](/docs/configuration/#watch) command.

    ```bash title="Terminal"
    pnpx yummacss build
    ```
  </Step>
</Stepper>

## Dependency changes

Yumma CSS now functions as a CLI tool. Install it as a dev dependency (`-D`) instead of a standard dependency to keep your production bundle lean.

```bash title="Terminal"
pnpm add yummacss -D
```

### Core package

Replace the `@yummacss/api` package with `@yummacss/core`. Update your imports accordingly.

```bash title="Terminal"
pnpm remove @yummacss/api
pnpm add @yummacss/core
```

### Color utility changes

In the current version, light (`l-`) & dark (`d-`) characters are removed across all color utilities. `lead` is renamed to `slate`.

<div className="p-4">
  ```html del={1} ins={2} title="index.html"
  <button class="bg-l-lead-6 h:bg-d-lead-1">Button</button>
  <button class="bg-slate-1 h:bg-slate-7">Button</button>
  ```
</div>

Color hue modification increased from 10 % to 14 %. Light colors are lighter; dark colors are darker.

| Legacy         | Latest        |

## Base style changes

Modern Yumma CSS versions include standardized base styles for better consistency. These are enabled by default & can be disabled in `yumma.config.mjs`.

Remove paddings by default:

```css mark={3} title="base.ts"
* {
  margin: 0;
  padding: 0;
}
```

Smother font rendering with a default system font. Source: [joshwcomeau.com](https://joshwcomeau.com/css/custom-css-reset).

```css mark={2} title="base.ts"
html {
  font-family: system-ui, sans-serif;
}
```

Form elements now include padding by default. Borders are added for form elements without class attributes.

```css ins={2-3} del={4} title="base.ts"
body {
  -webkit-font-smoothing: antialiased;
  font-family: system-ui, sans-serif;
  line-height: 1.5;
}
```

Form elements now include padding by default. Borders are added for form elements without class attributes.

```css mark={6-8,16} title="base.ts"
button,
input,
optgroup,
select,
textarea {
  background-color: transparent;
  font-family: inherit;
  padding: 0.5 rem;
}

button:not([class]),
input:not([class]),
optgroup:not([class]),
select:not([class]),
textarea:not([class]) {
  border: 1px solid #bfc2c7;
}
```

Clear outlines for interactive elements to improve accessibility.

```css mark={7-9} title="base.ts"
button,
input,
textarea,
select,
a,
summary {
  &:focus {
    outline: 2px solid transparent;
  }
}
```

Textareas without `rows` use a default height. Source: [piccalil.li](https://piccalil.li/blog/a-more-modern-css-reset).

```css mark={2} title="base.ts"
textarea:not([rows]) {
  min-height: 10 em;
}
```

Distinct visual states for disabled elements.

```css mark={5-6} title="base.ts"
button:disabled,
input:disabled,
select:disabled,
textarea:disabled {
  cursor: not-allowed;
  opacity: 0.5;
}
```

Headings use balanced wrapping, consistent sizes, & bold weights.

```css mark={7-9,13} title="base.ts"
h1,
h2,
h3,
h4,
h5,
h6 {
  font-size: 1rem;
  font-weight: 600;
  text-wrap: balance;
}

p {
  text-wrap: pretty;
}
```

Standardized scaling for small text & code elements. Source: [modern-normalize](https://github.com/sindresorhus/modern-normalize).

```css mark={3,7-8,15-16} title="base.ts"
b,
strong {
  font-weight: 700;
}

small {
  font-size: 80 %;
  line-height: 1.4;
}

pre,
code,
kbd,
samp {
  font-family: monospace, monospace;
  font-size: 1 em;
}
```

Reset default link styles:

```css mark={2,7} title="base.ts"
a {
  color: inherit;
  text-decoration: none;
}
```

Consistent table header styling:

```css mark={2-3} title="base.ts"
th {
  font-size: 1rem;
  font-weight: 600;
}
```

Updated spacing for miscellaneous elements.

```css mark={3,8,12} title="base.ts"
hr {
  border-top: 1px solid #bfc2c7;
  height: 0;
  margin: 1 em 0;
}

details {
  display: block;
}

summary {
  display: list-item;
}
```

### Disable base styles

Set `buildOptions.reset` to `false` in `yumma.config.mjs` to disable base styles.

```js mark={5} title="yumma.config.mjs"
export default {
  buildOptions: {
    reset: false,
    minify: false,
  },
};
```
