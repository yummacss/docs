Maintaining and scaling styles in CSS can be a lot of work, whether you're using [inline or external styles](https://www.w3schools.com/html/html_css.asp). It can be really frustrating when you're trying to keep your designs consistently predictable, and most importantly, scalable.

{/* excerpt */}

## What is Yumma CSS?

Think of Yumma CSS as a set of abbreviated utility classes for building faster and more maintainable UIs. [Yumma CSS](/) has a wide range of reusable utilities that you can apply to your design system to make it more consistent and less unpredictable.

## Using Flexbox utilities

Let’s look at a classic layout case: centering with flexbox.

### Using inline styles

```html
<div style="align-items: center; display: flex; justify-content: center;">...</div>
```

### Using external styles

```css
p {
  align-items: center;
  display: flex;
  justify-content: center;
}
```

**Using Yumma CSS**

```html
<div class="ai-c d-f jc-c">...</div>
```

You end up with the same result, but the code is much shorter, more reusable, and way easier to grow.

You can use consistent patterns like `m-{n}` or `bg-{color}` to get faster prototyping directly in markup without writing the traditional inline or external styles.

## Overriding styles conditionally

So, let's say you use a specific utility in your markup and you want to override its value based on the user's [screen size](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_media_queries/Using_media_queries) or other factors, such as [hover states](https://developer.mozilla.org/en-US/docs/Web/CSS/:hover).

Yumma CSS hooks you up with [utility modifiers](/docs/foundations/responsive-design) that let you target use cases like `sm:*`, `md:*`, `lg:*`, and `xxl:*` to target a specific viewport, and `h:*` to target hover states.

## Media queries

Let's take a look at how we can use media queries to change the layout of a grid based on the size of the screen.

### Using external styles

```css
.container {
  display: grid;
  gap: 4rem;
  grid-template-columns: 1fr;
}

@media (min-width: 640px) {
  .container {
    grid-template-columns: repeat(3, 1fr);
  }
}
```

### Using Yumma CSS

```html
<div class="d-g g-16 gtc-1 sm:gtc-3">
  <div class="bg-indigo d-16"></div>
  <div class="bg-indigo d-16"></div>
  <div class="bg-indigo d-16"></div>
</div>
```

## Hover states

Also, you can control [hover states](/docs/foundations/variants/) with Yumma CSS. For example, if you want to hide an element when the user hovers over it, you can use the `h:*` modifier.

### Using external styles

```css
.flex {
  display: flex;
}

.hidden:hover {
  display: none;
}
```

### Using Yumma CSS

```html
<div class="d-g g-16 gtc-1 sm:gtc-3">
  <div class="ai-c bg-indigo d-16 d-f h:d-none jc-c"></div>
</div>
```

## Getting started

Try [Yumma CSS with npm](https://www.npmjs.com/package/yummacss) or in your browser on [Yumma CSS Play](https://play.yummacss.com).

```bash
npm install yummacss -D
```
