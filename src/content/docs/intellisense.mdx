## Requirements

Add a `yumma.config.mjs` file at your workspace root to activate the extension. It reloads automatically when the file changes.

## Language Server

Any LSP-compatible editor can run the Yumma CSS language server directly:

```bash
npx @yummacss/language-server --stdio
```

## Features

### Completion

Get auto-completion suggestions for every Yumma CSS utility class as you type, across HTML, JSX, TSX, Vue, Svelte, Astro, PHP & more.

### Hover Previews

View the exact CSS declaration behind any class by hovering over it.

### Color Decorators

See inline color swatches next to color utilities to spot the palette at a glance.

### Conflict Detection

Get real-time warnings when multiple classes set the same CSS property. For example, `bg-indigo bg-red` triggers a warning because both utilities set `background-color`. Conflicts are scoped per-variant, so `bg-red h:bg-indigo` doesn't conflict since they apply under different conditions.

### Quick Fixes

Resolve conflicts with 1-click quick fixes.

### Class Sorting

Keep utility classes in a consistent, readable order automatically.

#### Sorting Order

Classes are organized by category: Layout &rarr; Positioning &rarr; Flexbox/Grid &rarr; Box Model &rarr; Background &rarr; Colors &rarr; Border/Outline &rarr; Typography &rarr; Effects &rarr; Transition &rarr; Interactivity &rarr; Table &rarr; SVG.

#### Variant Grouping

Variant classes like `h:bg-red` stay grouped with their base utility instead of sorting independently. Orphan variants (those with no matching base class) move to the end of the class list.

#### Triggering a Sort

In VS Code, classes sort on save by default. Toggle this with the `yummacss.sortOnSave` setting, or run `Yumma CSS: Sort Classes` from the Command Palette (`Ctrl+Shift+P`) to sort manually.

In Zed, run `editor: format` from the Command Palette.
