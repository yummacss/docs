Building apps with vanilla CSS can be a lot of work, but it doesn't have to be. Yumma CSS uses an abbreviated naming convention for its utility classes, which makes it easier to scale and maintain with less code, faster debugging, and more readable and understandable code.

## Using vanilla CSS

If you use regular, you'll end up with larger files, which will take longer to split and will be harder to maintain and scale.

```html live
<style>
  .btn-indigo {
    background-color: #595cd9;
    color: white;
    border-radius: 2rem;
    border-width: 0px;
    padding-left: 1.25rem;
    padding-right: 1.25rem;
    padding-bottom: 0.25rem;
    padding-top: 0.25rem;
    cursor: pointer;
    font-weight: 600;
  }

  .btn-indigo:hover {
    background-color: #474aae;
  }
</style>

<button class="btn-indigo">Subscribe</button>
```

---

## Using Yumma CSS

Yumma CSS uses abbreviated naming convention, which will make the files smaller. This will make it easier to maintain and scale applications without having to learn a whole new markup.

{/* prettier-ignore */}
```html live
<button class="b-0 bg-indigo fw-600 h:bg-d-indigo-2 px-5 py-1 rad-8 tc-white c-p">
  Subscribe
</button>
```
