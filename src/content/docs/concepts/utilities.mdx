Building apps with vanilla CSS can be a lot of work, but it doesn't have to be. Yumma CSS uses an abbreviated naming convention for its utility classes, which makes it easier to scale and maintain with less code, faster debugging, and more readable and understandable code.

<Aside type="danger" title="Using vanilla CSS">
  If you use regular, you’ll end up with larger files, which will take longer to
  split and will be harder to maintain and scale.
</Aside>

---

<Aside type="tip" title="Using Yumma CSS">
  Yumma CSS uses abbreviated naming convention, which will make the files
  smaller. This will make it easier to maintain and scale applications without
  having to learn a whole new markup.
</Aside>
